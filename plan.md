1. **Remove `panic()` on untrusted input lengths**: The `ReadBytes` and `ReadStringArray` functions in `moqt/internal/message/message_reader.go` use `panic()` when parsing varints that exceed `math.MaxInt`. This is a classic Denial of Service (DoS) vector if an attacker supplies extremely large length values, crashing the application.
2. **Prevent OOM (Out Of Memory) DoS in `ReadStringArray`**: In `ReadStringArray`, a slice of strings is allocated with a capacity equal to the unvalidated `count` read from the varint. If `count` is very large but under `math.MaxInt`, `make([]string, 0, count)` will allocate an excessive amount of memory, causing OOM. I will add a check that `count > uint64(len(b))` and return `io.EOF` if so, as each element takes at least 1 byte (the length varint itself), safely bounding the allocation and supporting chunked reads.
