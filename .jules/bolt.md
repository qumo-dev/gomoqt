## 2024-05-24 - Zero-Allocation Varint Reading
**Learning:** In `gomoqt`, network stream readers might process a massive amount of small messages. The `ReadMessageLength` function was originally reading bytes into a slice using `io.ReadFull(r, firstByte)`, which triggered escape analysis because `firstByte` was allocated via `make([]byte, 1)`. Even if pre-allocated locally as `var firstByte [1]byte; io.ReadFull(r, firstByte[:])`, passing a slice of a stack array into an interface method like `io.Reader.Read` generally escapes it to the heap.
**Action:** Always optimize hot path readers (like varints) by using a fast-path for `io.ByteReader` directly, mapping `io.EOF` to `io.ErrUnexpectedEOF` on subsequent reads, and falling back to stack-allocated arrays for generic `io.Reader` without pointers escaping to interfaces if possible.
