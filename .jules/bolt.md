## 2024-06-09 - QUIC Varint parsing hot-path allocation penalty
**Learning:** In `gomoqt`, using `io.ReadFull(r, buf[:])` or `firstByte := make([]byte, 1); io.ReadFull(r, firstByte)` inside hot-path varint parsing (like `ReadMessageLength`) causes the buffer slice to escape to the heap due to escape analysis limits, incurring a significant allocation penalty for every varint read.
**Action:** Always implement a fast-path for `io.Reader` using type assertion `r.(io.ByteReader)`. Reading bytes sequentially using `ReadByte()` avoids slice passing entirely, keeping parsing on the stack and eliminating allocation overhead on the hot path. Remember to correctly map `io.EOF` to `io.ErrUnexpectedEOF` after the first byte when using `ReadByte()` to match standard stream semantics.
