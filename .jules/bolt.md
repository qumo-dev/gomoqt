## 2024-05-17 - Stack Arrays Escape on Interface Boundaries
**Learning:** In Go, replacing `make([]byte)` with a slice of a local stack array (e.g., `buf[:1]` from `var buf [8]byte`) does not completely eliminate heap allocations when the slice is passed to an interface method like `io.Reader.Read` or `io.ReadFull`. The compiler's escape analysis is forced to move the underlying array to the heap because it cannot guarantee the interface implementation won't retain a reference to the slice.
**Action:** When optimizing interface fallback paths in `io.Reader` wrappers, local stack arrays can consolidate multiple allocations into a single, smaller allocation, but attempting to reach 0 allocs/op may require deeper architectural changes (like full `sync.Pool` integration or restructuring the reader logic) rather than simple variable swaps.
