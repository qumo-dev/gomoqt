## 2026-06-07 - Denial of Service via Slice Allocation and Panic in Message Parser
**Vulnerability:** The message parser functions `ReadBytes` and `ReadStringArray` were vulnerable to Denial of Service (DoS) attacks. `ReadBytes` and `ReadStringArray` paniced when the decoded varint size or count exceeded `math.MaxInt`. More critically, `ReadStringArray` used `make([]string, 0, count)` without validating if the decoded count was physically possible given the remaining bytes in the buffer, leading to OOM (Out-Of-Memory) via excessive slice allocation.
**Learning:** Panics on untrusted network payloads break service availability. Pre-allocating slice capacities based purely on untrusted varints without bounding them against the actual buffer size is a severe DoS vector.
**Prevention:** Always replace panics with standard error returns (e.g., `errors.New`, `io.EOF`) in payload parsing. Always validate array/slice allocation lengths (`count`) against the available data size (`uint64(len(b))`) before using them in `make()`.
